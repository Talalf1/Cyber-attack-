📌 Missing Components — Detailed Overview
📦
VirtualBox Integration
Infrastructure Layer · Phase 2

The current website operates as a standalone web application. The full project architecture requires integration with VirtualBox to manage isolated virtual machines for attack simulations. This includes VM snapshot management, network isolation, and automated provisioning via APIs.

VirtualBox API VM Snapshots Network Isolation Auto Provisioning
🐧
Kali Linux Attacker Machine
Attack Engine · Phase 2

The project proposal includes a dedicated Kali Linux virtual machine used to simulate real attack scenarios such as network scanning, exploitation, and privilege escalation within a controlled isolated environment. Tools include Metasploit, Nmap, Hydra, and SQLmap.

Metasploit Nmap Hydra SQLmap Wireshark
🖥️
Target Systems (Win & Linux)
Target Environment · Phase 2

Windows and Linux virtual machines should be deployed as target systems to generate realistic attack and defense scenarios. These systems would host vulnerable services intentionally for educational exploitation, modeled after platforms like DVWA and Metasploitable.

Windows Server Ubuntu Linux DVWA Metasploitable
🔍
Wazuh SIEM Integration
Monitoring Layer · Phase 3

The current version does not fully integrate Wazuh for log collection, alert generation, and security event monitoring. Future versions should display real-time alerts and attack logs pulled directly from Wazuh agents deployed on target virtual machines.

Wazuh Agent Log Collection Alert Rules SIEM Dashboard REST API
📊
Real-Time Log Analysis
Analytics Layer · Phase 3

The dashboard currently focuses on educational simulations. Future development should include actual real-time security logs from deployed VMs, event correlation engines, attack pattern detection, and a full SIEM-style monitoring interface.

Event Correlation Attack Patterns Live Streaming Anomaly Detection
⚙️
Attack Simulation Automation
Automation Engine · Phase 3

The website currently provides educational attack simulations only. Future versions can automate real simulations between attacker and target virtual machines — including scheduled attacks, scenario scripting, and full kill-chain automation — while maintaining a completely secure and isolated environment.

Script Engine Scheduled Attacks Kill Chain Isolation Layer
📋
Reporting & Analytics
Reporting Layer · Phase 4

Additional reporting features can be added, including full attack history timelines, security metrics and KPIs, user performance tracking across sessions, and downloadable PDF/CSV reports suitable for academic and professional submissions.

PDF Export Attack History Security KPIs Performance Metrics
🏆
User Progress Tracking
Gamification Layer · Phase 4

Future versions may include advanced user scoring systems, achievement and badge collections, detailed training progress tracking across all modules, leaderboards, and official certification support for completing cybersecurity training tracks.

Leaderboards Certifications XP System Progress Reports
